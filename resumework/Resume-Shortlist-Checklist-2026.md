# Resume Shortlist Checklist (Senior Java Full Stack)

Use this checklist before every application to reduce rejection risk.

## 0) Corrected Resume Versions in `resumework`
- `Ram-Kotni-Resume-ATS-2026.md` (general ATS-friendly version)
- `Ram-Kotni-Resume-Java-FullStack-Lead-2026.md` (primary Java full-stack roles)
- `Ram-Kotni-Resume-Python-FullStack-Lead-2026.md` (Python + Java roles)
- `Ram-Kotni-Resume-AgilePLM-Architect-2026.md` (Agile PLM/integration architect roles)

Use one primary version per application and customize only that version for the target job.

## 1) ATS and Formatting
- Keep resume to 1.5 to 2 pages for 15+ years profile.
- Use standard headings: Summary, Skills, Experience, Education, Certifications.
- Avoid tables, icons, images, and special characters that ATS cannot parse.
- Keep date format consistent: `Mon YYYY - Mon YYYY`.
- Use one email, one phone, and one LinkedIn URL consistently.

## 2) Role Targeting
- Keep one primary target title per resume version:
  - Senior Java Full Stack Engineer
  - Lead Full Stack Engineer
  - Application Architect
- Mirror keywords from the job description in your summary and top 2 roles.
- Reorder skills to match role priority (for example Java/Spring/AWS first).

## 3) Bullet Quality Rule (must follow)
Each bullet should include:
- **Action** (designed, led, optimized, built)
- **Scope** (module, workflow, number of APIs/services)
- **Tech** (Java, Spring Boot, Angular, Kafka, AWS)
- **Impact** (latency reduction, uptime, faster release, cost reduction)

Example:
- `Designed and delivered 10+ Spring Boot APIs for resource workflows, improving reliability and reducing manual processing effort.`

## 4) Common Rejection Causes to Avoid
- Too many tools listed without evidence in experience.
- Generic bullets (`worked on`, `involved in`) without outcomes.
- Inconsistent years of experience (16 in one place, 18 in another).
- Overloaded older-role details while recent roles are not deep enough.
- Including sensitive personal details (DOB, ID documents, references) in resume.

## 5) 20-Second Recruiter Scan Test
Top half of page 1 must clearly answer:
- Who are you? (senior Java full-stack)
- What stack? (Java, Spring Boot, Angular/React, AWS, Kafka)
- What outcomes? (reliability, scale, delivery, modernization)

## 6) Customization Workflow for Each Job (10 minutes)
1. Copy JD keywords.
2. Update summary with 4-6 matching keywords.
3. Update top 8 bullets in latest 2 roles with JD language.
4. Ensure same keywords appear in skills and experience.
5. Save as `Role-Company-YYYY-MM.md`.

## 7) Interview Alignment
- Resume claims must match your STAR stories.
- Keep 3 prepared stories: production incident, architecture decision, leadership impact.
- Keep 3 metrics ready: performance, reliability, release speed.

## 8) Suggested Resume Versions
- `Java + Angular + AWS` (primary)
- `Java + Microservices + Platform Reliability/SRE`
- `Java + Architect/Lead` (for principal/architect postings)

## 9) Final Pre-Submit Check
- No typos, no grammar errors, no special-character artifacts.
- PDF export verified.
- Filename professional: `Ram_Kotni_Senior_Java_FullStack_Resume.pdf`.
- LinkedIn and resume headline aligned.

## Note
No resume can guarantee zero rejection, but this checklist significantly improves shortlist probability when combined with targeted applications and referral-based submissions.
